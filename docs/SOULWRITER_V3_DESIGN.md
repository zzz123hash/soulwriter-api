# SoulWriter v3 系统设计文档

## 一、核心理念

**SoulWriter = 跨文化故事迁移引擎**

不是翻译工具，而是：
- 提取故事线（角色、物品、地点、事件）
- 迁移文化背景（中国→西方、日本→奇幻等）
- 保持故事线不变，文化元素可以适配
- AI可以丰富内容（可选）

---

## 二、核心数据结构

### 2.1 节点 (Node) - 最小单位

节点是剧本的"一幕"，包含：

```javascript
Node = {
  id: string,
  type: "role" | "item" | "location" | "event",
  name: string,
  content: string,           // 片段内容（剧本式描写）
  time?: string,            // 发生时间
  place?: string,           // 发生地点
  characters?: string[],     // 涉及角色
  items?: string[],         // 涉及物品
  emotion?: string,          // 情绪基调
  perspective?: string,      // 视角
  metadata: {
    culture: string,        // 文化背景
    createdAt: string,
    updatedAt: string
  }
}
```

### 2.2 片段 (Fragment) - 一个节点 = 剧本的"一幕"

片段是节点的实例化：
- 一个片段 = 一个节点 + 时间线位置 + 上下文

### 2.3 单元 (Unit) - 章节

单元 = 多个片段拼接 = 一个章节

```javascript
Unit = {
  id: string,
  title: string,
  fragments: Fragment[],     // 片段列表（按时间线排列）
  perspective: string,       // 主视角角色ID
  perspectiveScore: number,  // 视角评分(0-100)
  settings: {
    dialogueRatio: number,      // 对话比例 0-100
    sceneryRatio: number,        // 风景描写比例 0-100
    emotionRatio: number,        // 情感描写比例 0-100
  },
  status: "draft" | "polished" | "final",
  createdAt: string,
  updatedAt: string
}
```

## 三、功能模块

### 3.1 创世树 (Genesis)

**功能**: 世界观结构规划

- 种子(Seed) → 节点(Node) → 关系树
- 节点类型: root/plot/role/item/location/world/climax/end
- 可视化: D3力导向图

### 3.2 女娲推演 (Nvwa) - 可选

**功能**: AI丰富内容

女娲不是必须的，作用是：
- 保持角色一致性
- 生成对话和互动
- 丰富文章内容

**触发条件**: 拖动单元到女娲区域

### 3.3 单元编辑 (Unit Editor)

**功能**: 组装节点成章节

- 拖拽节点到单元
- 查看每个片段详情
- 二次编辑节点
- 查看片段的时间、地点、人物、事件

### 3.4 视角转写 (Perspective Rewrite)

**功能**: 切换主角视角

**规则**:
- 不是所有角色都能当主角
- AI评分系统评估角色是否适合

**评分维度**:
| 维度 | 说明 |
|------|------|
| 素材量 | 角色出现的次数、描写量 |
| 记忆占比 | 角色在女娲中的记忆量 |
| 对话量 | 角色对话占比 |
| 冲突参与 | 参与核心冲突的次数 |
| 人物弧线 | 是否有成长变化 |

**转写条件**:
- **>80分** → 可以转写第一人称
- **<80分** → 第三人称 or 同人文
- 选中角色后AI给出可转写建议

**示例**:
```
盗墓笔记三人组:
- 吴邪 → 90分 ✓ 第一人称
- 胖子 → 75分 ✓ 第三人称可转
- 张起灵 → 45分 ✗ 素材太少，同人文
```

### 3.5 小说编辑器 (Novel Editor)

**功能**: 拖拽单元生成小说

**入口**: 将单元拖入小说页面

**设置参数**:
| 参数 | 说明 |
|------|------|
| 对话比例 | 0-100 |
| 风景描写比例 | 0-100 |
| 情感描写比例 | 0-100 |

**AI参考**:
- 人物形象
- 人物记忆
- 故事线
- 预设

**输出**: 性价比高、质量上乘的文章

**可修改**:
- 圈出内容让AI重新修改
- 或一键重写

### 3.6 文化转写 (Culture Rewrite)

**功能**: 跨文化背景迁移

**设计原则**:
- 不是文化清洗，而是文化适配
- 人名地名可保留或音译
- 文化特色词需要解释性翻译

**强度级别**:
| 强度 | 行为 |
|------|------|
| 0-20% | 基础翻译，只翻文字 |
| 20-40% | 文化优化，新年→圣诞节，调整语序 |
| 60-80% | 深度本土化，修仙→魔法，皇帝→领主 |
| 80-100% | 完全重写，只保故事线 |

**文化预设库**:
| 源文化 | 目标文化 | 映射 |
|--------|----------|------|
| 中国武侠 | 西方奇幻 | 皇帝→领主、镖局→佣兵公会、轻功→飞行术 |
| 中国仙侠 | 西方魔法 | 筑基→觉醒、元婴→大法师、飞剑→魔法剑 |
| 日本战国 | 中东宫廷 | 大名→苏丹、忍者→刺客、幕府→哈里发 |

## 五、技术实现要点

### 5.1 数据模型

```javascript
// 角色评分
RoleScore = {
  roleId: string,
  materialScore: number,     // 素材量 0-100
  memoryScore: number,         // 记忆占比 0-100
  dialogueScore: number,       // 对话量 0-100
  conflictScore: number,       // 冲突参与 0-100
  arcScore: number,           // 人物弧线 0-100
  totalScore: number,          // 总分 0-100
}

// 片段 = 节点实例
Fragment = {
  id: string,
  nodeId: string,
  position: number,            // 时间线位置
  context: string,             // 上下文
}

// 单元 = 章节
Unit = {
  id: string,
  title: string,
  fragments: Fragment[],
  perspective: string,         // 主视角角色ID
  perspectiveScore: number,   // 视角评分
  settings: UnitSettings,
  history: HistoryEntry[],    // 修改历史
}
```

### 5.2 AI Prompt 设计

**角色评分Prompt**:
```
分析以下角色在小说中的素材积累程度：
角色名：{name}
出场次数：{count}
描写量：{description}
对话量：{dialogue}
参与冲突：{conflicts}

请评估：
1. 素材量评分(0-100)
2. 记忆占比评分(0-100)
3. 对话量评分(0-100)
4. 冲突参与评分(0-100)
5. 人物弧线评分(0-100)

总分大于80才可以转写第一人称视角。
```

**视角转写Prompt**:
```
将以下章节从{A}的第一人称视角转写为{B}的第一人称视角。
{B}的素材积累程度：{score}/100
如果{B}素材不足80分，请转写为第三人称。

章节内容：
{content}

要求：
1. 保持故事线不变
2. {B}如果有素材，保持其内心独白和视角
3. 如果素材不足，用第三人称旁观视角
```
